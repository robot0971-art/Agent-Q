# AgentQ CLI 실행 방법

## 1. 환경변수 설정 (권장)

### Windows CMD
```cmd
set AGENTQ_PROVIDER=openai
set AGENTQ_MODEL=kimi-k2.5
set AGENTQ_BASE_URL=https://coding-intl.dashscope.aliyuncs.com/v1
set AGENTQ_API_KEY=sk-sp-cd0a1f191a6e422b831d3aaa9e2227f6
set AGENTQ_WORKSPACE_ROOT=C:\Users\admin\Desktop\Agent-Q
```

### PowerShell
```powershell
$env:AGENTQ_PROVIDER="openai"
$env:AGENTQ_MODEL="kimi-k2.5"
$env:AGENTQ_BASE_URL="https://coding-intl.dashscope.aliyuncs.com/v1"
$env:AGENTQ_API_KEY="sk-sp-cd0a1f191a6e422b831d3aaa9e2227f6"
$env:AGENTQ_WORKSPACE_ROOT="C:\Users\admin\Desktop\Agent-Q"
```

## 2. CLI 실행

```cmd
cd C:\Users\admin\Desktop\Agent-Q\csharp
dotnet run --project AgentQ.Cli
```

## 3. 명령행 인수로 직접 설정

환경변수 없이 직접 인수로 설정:

```cmd
dotnet run --project AgentQ.Cli -- --provider openai --model kimi-k2.5 --base-url https://coding-intl.dashscope.aliyuncs.com/v1 --api-key sk-sp-cd0a1f191a6e422b831d3aaa9e2227f6
```

## 4. 실행 확인

정상 실행 시 다음 메시지가 표시됩니다:

```
AgentQ CLI
Provider: openai
Model: kimi-k2.5
Base URL: https://coding-intl.dashscope.aliyuncs.com/v1
Tools registered: 7
Type /help for commands.

>
```

## 5. 지원하는 모델 (Alibaba Coding Plan)

- `kimi-k2.5` - 기본 권장 모델
- `qwen-coder-plus` - Qwen 코더 모델
- `qwen-max` - Qwen 최대 모델
- `qwen-plus` - Qwen 플러스 모델

## 6. CLI 명령어

대화 시작 후 사용 가능한 명령어:

- `/help` - 도움말 표시
- `/model <모델명>` - 모델 변경
- `/provider <제공자>` - 제공자 변경
- `/base-url <URL>` - API 베이스 URL 변경
- `/exit` 또는 `/quit` - 종료

## 7. 문제 해결

### 401 Unauthorized 오류
- API 키가 올바른지 확인
- Coding Plan API 키는 `sk-sp-`로 시작

### 404 Not Found 오류
- Base URL 끝에 `/`가 없는지 확인
- 올바른 URL: `https://coding-intl.dashscope.aliyuncs.com/v1`

### 프로세스 충돌 (빌드 오류)
```cmd
taskkill /F /IM AgentQ.Cli.exe
```

## 참고사항

- **API 키 보안**: API 키를 코드나 공개 저장소에 저장하지 마세요
- **하위 호환성**: `CLAW_` 접두사 환경변수도 여전히 지원됩니다
- **모델 설정**: Alibaba Coding Plan은 `kimi-k2.5` 모델을 사용합니다
